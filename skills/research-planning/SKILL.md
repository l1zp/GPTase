# Research Planning

## Overview

提供系统的研究任务规划方法论。将复杂任务分解为可执行的阶段，每个阶段有明确的输入、输出和验证标准。

## When to Use

- 任务涉及多个步骤或多种数据源
- 需要先调研再执行
- 用户明确要求"规划"或"制定方案"

## Planning Phases

### Phase 1: 需求分析

1. **解析任务**：从用户输入中提取关键信息
   - 研究对象（酶名、反应类型、基因等）
   - 目标输出（数据表、报告、对比分析等）
   - 约束条件（时间范围、数据源限制等）

2. **背景调研**：使用 WebSearch 快速了解领域现状
   - 搜索 2-3 个关键词，获取概览
   - 识别可用的数据源和数据库

3. **输出**：需求清单 + 已有数据概览

### Phase 2: 方案设计

1. **确定数据源**：列出需要查询的数据库/API
   - 使用 `biochem_databases` skill 中的 API 端点
   - 确定查询参数和过滤条件

2. **设计工作流**：按依赖关系排序执行步骤

   ```
   步骤示例：
   1. 查询 OpenAlex 获取相关论文
   2. 下载 PDF 并用 academic-pdf-reader 提取数据
   3. 查询 PDB 获取蛋白质结构
   4. 查询 KEGG 获取代谢通路
   5. 整合数据并生成报告
   ```

3. **输出**：执行计划（步骤列表 + 预期输出格式）

### Phase 3: 执行

按计划逐步执行，每步完成后：

1. **验证结果**：检查输出格式和完整性
2. **记录进展**：更新已完成步骤
3. **调整计划**：根据中间结果调整后续步骤

### Phase 4: 汇总

1. **整合数据**：合并各步骤的输出
2. **质量检查**：
   - [ ] 所有必需数据是否已获取？
   - [ ] 数据格式是否一致？
   - [ ] 是否有缺失或矛盾？
3. **生成最终报告**

## Quality Standards

- 每个数据点应有来源标注
- 数值数据应包含单位
- 如有不确定性，明确标注
- 如某步骤失败，记录原因并尝试替代方案

## Common Patterns

### 酶学研究
```
OpenAlex (论文) → PDF Reader (提取) → PDB (结构) → KEGG (通路) → 报告
```

### 文献综述
```
OpenAlex (搜索) → PDF Reader (逐篇分析) → 对比表格 → 综述报告
```

### 蛋白质分析
```
UniProt (序列) → PDB (结构) → ExPASy (酶信息) → Rhea (反应) → 总结
```
